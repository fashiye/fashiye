## 通用说明

### 认证方式

所有受保护的接口需在请求头中携带 Bearer Token：

```
Authorization: Bearer <access_token>
```

### 角色与权限

| 角色标识 | 说明 | 权限等级 |
|:---|:---|:---|
| `user` | 普通用户（下单方） | 低 |
| `handler` | 打手（接单方） | 中 |
| `operator` | 运营管理员 | 高 |
| `super` | 超级管理员 | 最高 |

### 通用响应格式

**成功响应（常规）：**
```json
{
  "code": 0,
  "data": { ... },
  "message": "操作成功"
}
```

**成功响应（列表/分页）：** 直接返回数据数组或分页对象。

**错误响应：**
```json
{
  "code": 400,
  "message": "错误描述信息",
  "data": null
}
```

### HTTP 状态码说明

| 状态码 | 说明 |
|:---|:---|
| 200 | 请求成功 |
| 400 | 业务错误（参数校验失败、业务规则冲突） |
| 401 | 未认证（Token 缺失或无效） |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如订单已被抢） |
| 500 | 服务器内部错误 |

### 分页参数约定

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|:---|:---|:---|:---|:---|
| `page` | int | 1 | >= 1 | 页码 |
| `size` | int | 20 | [1, 100] | 每页条数 |

## 1. 认证模块（Auth）

### `POST /api/v1/auth/send-code`

发送邮箱注册验证码。

**权限：** 公开

**请求体：**
```json
{
  "email": "user@example.com"
}
```

**响应：**
```json
{
  "message": "验证码已发送"
}
```

---

### `POST /api/v1/auth/register`

用户注册。

**权限：** 公开

**请求体：**
```json
{
  "username": "用户名",
  "email": "user@example.com",
  "password": "密码（至少6位，含字母+数字）",
  "phone": "13800138000",
  "role": "user | handler | admin",
  "verification_code": "123456"
}
```

| 字段 | 说明 |
|:---|:---|
| `role` | 注册角色。`user`=普通用户，`handler`=打手，`admin`=管理员 |
| `phone` | 仅 `user` 和 `handler` 需要 |
| `verification_code` | 通过 `send-code` 接口获取的验证码 |
| `password` | 必须包含字母和数字，长度至少 6 位 |

**响应：** `TokenResponse`
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "用户名",
    "email": "user@example.com",
    "avatar": null,
    "status": 1,
    "role": "user",
    "phone": "13800138000"
  }
}
```

---

### `POST /api/v1/auth/login`

用户登录。

**权限：** 公开

**请求体：**
```json
{
  "email": "user@example.com",
  "password": "密码",
  "role": "user | handler | admin"
}
```

| 字段 | 说明 |
|:---|:---|
| `role` | 选择以何种角色登录。admin 角色可在数据库中进一步分为 `super` 或 `operator` |

**响应：** `TokenResponse`（同注册响应）

**特殊状态码：**
- `403` — 打手账号需审核（status=0）、审核未通过（status=2）或被禁用（status=3）

---

### `POST /api/v1/auth/forgot-password`

发送密码重置验证码。

**权限：** 公开（频率限制：每 5 分钟最多 3 次）

**请求体：**
```json
{
  "email": "user@example.com",
  "role": "user | handler | admin"
}
```

**响应：**
```json
{
  "message": "密码重置验证码已发送"
}
```

---

### `POST /api/v1/auth/reset-password`

重置密码。

**权限：** 公开（频率限制：每 5 分钟最多 5 次）

**请求体：**
```json
{
  "email": "user@example.com",
  "role": "user | handler | admin",
  "verification_code": "123456",
  "new_password": "新密码（至少6位，含字母+数字）"
}
```

**响应：**
```json
{
  "message": "密码重置成功"
}
```

---

### `GET /api/v1/auth/me`

获取当前登录用户信息。

**权限：** 任何已认证用户

**请求头：** `Authorization: Bearer <token>`

**响应：** `UserResponse`
```json
{
  "id": 1,
  "username": "用户名",
  "email": "user@example.com",
  "phone": "13800138000",
  "avatar": null,
  "status": 1,
  "balance": 100.00,
  "level": 1,
  "total_orders": 10,
  "completion_rate": 100.00,
  "role": "handler"
}
```

> 注：`balance` 字段仅 `user` 和 `handler` 角色返回；`level`、`total_orders`、`completion_rate` 仅 `handler` 角色返回。

## 2. 订单模块（Orders）

### 订单状态流转

```
                  ┌─→ [pending_review] ─→ [pending] ─→ [accepted] ─→ [in_progress] ─→ [review] ─→ [completed]
                  │        (审核中)        (待接单)      (已接单)       (进行中)        (待验收)     (已完成)
[create] ────────┘
                  │
                  └─→ [cancelled] ←──────────── (取消)
                  └─→ [disputed]  ←──────────── (纠纷)
```

| 状态 | 说明 | 可见角色 |
|:---|:---|:---|
| `pending_review` | 待审核（订单创建后初始状态） | user, super, operator |
| `pending` | 待接单（管理员审核通过后进入订单池） | handler, super, operator |
| `accepted` | 已接单（打手接单） | user, handler |
| `in_progress` | 进行中（打手开始代练） | user, handler |
| `review` | 待验收（打手提交完成） | user, handler, super, operator |
| `completed` | 已完成（用户/管理员确认完成） | 所有 |
| `cancelled` | 已取消 | 所有 |
| `disputed` | 纠纷中 | super, operator |

### `POST /api/v1/orders`

创建订单。

**权限：** `user`

**请求体：**
```json
{
  "gameId": 1,
  "accountInfo": "账号密码信息（服务端会自动加密存储）",
  "requirements": "代练要求说明",
  "items": [
    {
      "projectId": 1,
      "quantity": 5
    },
    {
      "projectId": 2,
      "quantity": 10
    }
  ]
}
```

| 字段 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `gameId` | int | 是 | 游戏 ID |
| `accountInfo` | string | 是 | 账号信息（服务端加密存储） |
| `requirements` | string | 否 | 代练要求 |
| `items` | array | 是 | 至少一个项目，每个项目包含 `projectId` 和 `quantity` |

**响应：** `OrderResponse`

### `POST /api/v1/orders/{order_id}/cancel`

取消订单。

**权限：** 订单相关方（仅限 `pending` 状态可取消）

**路径参数：**

| 参数 | 类型 | 说明 |
|:---|:---|:---|
| `order_id` | int | 订单 ID |

**响应：** `OrderResponse`

---

### `POST /api/v1/orders/{order_id}/status`

更新订单状态（打手/用户操作）。

**权限：** 订单相关方

**路径参数：**

| 参数 | 类型 | 说明 |
|:---|:---|:---|
| `order_id` | int | 订单 ID |

**请求体：**
```json
{
  "action": "start | submit_complete | confirm_complete",
  "remark": "操作备注（可选）",
  "completionProof": ["完成截图URL1", "完成截图URL2"]
}
```

**状态操作说明：**

| action | 旧状态 | 新状态 | 操作者 | 说明 |
|:---|:---|:---|:---|:---|
| `start` | accepted | in_progress | handler | 开始代练 |
| `submit_complete` | in_progress | review | handler | 提交完成（需附完成凭证） |
| `confirm_complete` | review | completed | user | 确认完成 |
| `cancel` | pending | cancelled | 任一方 | 取消订单 |
| `dispute` | pending | disputed | 任一方 | 发起纠纷 |

**响应：** `OrderResponse`

---

### `POST /api/v1/orders/{order_id}/approve`

管理员批准订单（审核通过）。

**权限：** `super`, `operator`

**路径参数：**

| 参数 | 类型 | 说明 |
|:---|:---|:---|
| `order_id` | int | 订单 ID |

> 将订单从 `pending_review` 状态转换为 `pending`，进入订单池。

**响应：** `OrderResponse`

---

### `POST /api/v1/orders/{order_id}/reject`

管理员拒绝订单（审核不通过）。

**权限：** `super`, `operator`

**路径参数：**

| 参数 | 类型 | 说明 |
|:---|:---|:---|
| `order_id` | int | 订单 ID |

**查询参数：**

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `remark` | string | 否 | 拒绝原因 |

> 将订单从 `pending_review` 状态转换为 `cancelled`。

**响应：** `OrderResponse`

---

### `POST /api/v1/orders/{order_id}/rate`

评价订单。

**权限：** 订单中的 user 或 handler（已完成订单可评价）

**请求体：**
```json

  "role": "user | handler",

}
```

| 字段 | 类型 | 必填 | 限制 | 说明 |
|:---|:---|:---|:---|:---|
| `role` | string | 是 | `user` 或 `handler` | 评价者角色 |
| `rating` | int | 是 | 1~5 | 评分 |
| `comment` | string | 否 | — | 评价内容 |

**响应：**
```json
{
  "code": 0,
  "data": null,
  "message": "评价成功"
}
```

---

### `GET /api/v1/users/available`

获取可发起会话的用户列表（基于历史订单）。

**权限：** 任何已认证用户

> 根据当前用户角色，返回有过订单往来的对方用户列表（去重）：

| `admin` | user + handler | 所有历史订单中的双方用户 |

**响应：** `List[dict]`
```json
[
  {
    "id": 2,
    "username": "打手A",
    "role": "handler",
    "orderNo": "20260430123456",
    "orderInfo": "英雄联盟 - 排位赛代打"
  }
]
```

---

### `DELETE /api/v1/users/{user_id}`

删除用户（超级管理员专用）。

**权限：** `super`

**路径参数：**

| 参数 | 类型 | 说明 |
|:---|:---|:---|
| `user_id` | int | 用户 ID |

**查询参数：**

| 参数 | 类型 | 必填 | 说明 |
|:---|:---|:---|:---|
| `role` | string | 是 | 用户角色：`user`/`handler`/`admin` |
