# 游戏下单页面与凭据查询功能实现计划

## 需求分析

用户需要实现两个新功能：

### 功能一：游戏下单页面（game.fashiye.cn）
- 用户访问该域名时直接显示选游戏下单页面
- 无需登录即可下单（匿名下单）
- 下单完成后显示凭据号（订单号）
- 提供一键复制凭据号按钮
- 提示用户截图保存凭据

### 功能二：凭据查询页面（docs.fashiye.com）
- 用户输入凭据号后查询订单
- 自动跳转到订单详情页
- 订单详情页需要有与打手聊天的功能

## 技术方案

### 后端改动

1. **新增匿名下单接口**（无需认证）
   - 路径：`POST /api/v1/orders/anonymous`
   - 支持未登录用户创建订单
   - 返回订单号作为凭据

2. **新增凭据号查询接口**（无需认证）
   - 路径：`GET /api/v1/orders/lookup/{order_no}`
   - 通过订单号查询订单详情

3. **会话接口支持订单查询**（用于聊天功能）
   - 确保订单详情页能获取会话信息

### 前端改动

1. **新增简化下单页面**（GameOrderPage）
   - 游戏选择、项目选择、账号信息输入
   - 下单成功后显示凭据号和复制按钮
   - 提示截图保存

2. **新增凭据查询页面**（OrderLookup）
   - 凭据号输入框
   - 查询后自动跳转订单详情

3. **订单详情页增强**
   - 添加聊天组件
   - 支持与打手实时沟通

### 文件结构

```
frontend/src/
├── pages/
│   ├── GameOrderPage.jsx      # 游戏下单页面
│   ├── OrderLookup.jsx        # 凭据查询页面
│   └── OrderDetail.jsx        # 增强聊天功能
└── components/
    └── ChatPanel.jsx          # 聊天面板组件
```

## 实现步骤

### 步骤一：后端接口开发

1. **修改订单端点**（app/api/v1/端点/订单.py）
   - 新增 `匿名创建订单接口`
   - 新增 `根据订单号查询订单接口`

2. **修改会话端点**（app/api/v1/端点/会话.py）
   - 确保支持通过订单ID查询会话

### 步骤二：前端页面开发

1. **创建 GameOrderPage.jsx**
   - 游戏和项目选择
   - 表单验证
   - 下单成功后显示凭据号和复制功能

2. **创建 OrderLookup.jsx**
   - 凭据号输入
   - 查询订单
   - 自动跳转

3. **增强 OrderDetail.jsx**
   - 集成聊天面板组件
   - 支持发送消息

### 步骤三：路由配置

修改 `App.jsx`，添加新页面路由：
- `/game-order` - 游戏下单页面
- `/order-lookup` - 凭据查询页面

## 依赖与风险

### 依赖
- 后端已有订单创建逻辑
- 前端已有聊天组件基础

### 风险点
1. **匿名订单安全性**：需要验证请求来源
2. **凭据号泄露风险**：建议设置查询次数限制
3. **聊天功能权限**：需要验证用户是否为订单相关方

## 测试要点

1. 匿名用户能成功下单并获取凭据号
2. 凭据号能正确查询到订单
3. 复制按钮功能正常
4. 订单详情页聊天功能正常

- 前端页面：3小时
